# 并行翻译功能说明

## 📖 功能介绍

并行翻译是v3.0新增的性能优化功能，通过同时调用多个API请求来大幅提升翻译速度。

### 核心特性

- **智能模式切换**：自动根据文本数量选择串行或并行模式
- **动态并发控制**：根据实际任务量调整并发线程数
- **错误容错**：单个批次失败时自动降级处理
- **进度追踪**：实时显示翻译进度

## ⚡ 性能提升

根据测试数据统计：

| 文本数量 | 串行模式 | 并行模式 | 速度提升 |
|---------|---------|---------|---------|
| 10条    | 15秒    | 15秒    | 1.0x    |
| 50条    | 75秒    | 25秒    | 3.0x    |
| 100条   | 150秒   | 40秒    | 3.75x   |
| 200条   | 300秒   | 70秒    | 4.3x    |

**注意**：实际性能取决于API响应速度和网络状况。

## 🔧 配置说明

在 `.env` 文件中添加以下配置：

```env
# 并行翻译设置
MAX_WORKERS=6          # 最大并发线程数（建议4-8）
PARALLEL_ENABLED=true  # 是否启用并行翻译（true/false）
```

### 参数说明

#### MAX_WORKERS（最大并发线程数）

- **默认值**：6
- **推荐范围**：4-8
- **说明**：
  - 数值越大：并发越高，速度越快，但可能触发API限流
  - 数值越小：并发越低，更稳定，但速度提升有限
  - 建议根据API提供商的速率限制调整

#### PARALLEL_ENABLED（是否启用并行）

- **默认值**：true
- **可选值**：true / false
- **说明**：
  - `true`：启用并行翻译（推荐）
  - `false`：使用串行翻译（稳定但慢）

## 📊 工作原理

### 串行模式（传统方式）

```
[批次1] → 等待 → [批次2] → 等待 → [批次3] → 等待 → 完成
总耗时 = 批次1 + 批次2 + 批次3
```

### 并行模式（优化方式）

```
[批次1] ┐
[批次2] ├→ 同时处理 → 完成
[批次3] ┘
总耗时 ≈ max(批次1, 批次2, 批次3)
```

### 智能切换逻辑

1. **文本数量 ≤ 10**：使用串行模式（避免并行开销）
2. **文本数量 > 10 且 PARALLEL_ENABLED=true**：使用并行模式
3. **并发数动态调整**：min(MAX_WORKERS, 实际批次数)

## 💡 使用建议

### 场景1：网页翻译
- **推荐配置**：`MAX_WORKERS=6`，`PARALLEL_ENABLED=true`
- **理由**：网页通常有大量文本节点（100+），并行效果显著

### 场景2：文档翻译
- **推荐配置**：`MAX_WORKERS=4-6`，`PARALLEL_ENABLED=true`
- **理由**：文档段落数量适中，并行能有效提速

### 场景3：短文本翻译
- **推荐配置**：`PARALLEL_ENABLED=false` 或让系统自动选择
- **理由**：文本少时并行开销大于收益

### 场景4：API限流严格
- **推荐配置**：`MAX_WORKERS=2-3`，`PARALLEL_ENABLED=true`
- **理由**：降低并发避免触发限流

## ⚠️ 注意事项

### API速率限制

大多数API提供商有速率限制（如：60请求/分钟），过高的并发可能导致：
- 请求被拒绝（429 Too Many Requests）
- 账户被临时限制
- 翻译失败率增加

**建议**：
1. 查看API提供商的速率限制文档
2. 从较低的MAX_WORKERS（如4）开始测试
3. 根据实际效果逐步调整

### 翻译质量

并行翻译**不会**影响翻译质量：
- ✓ 每个批次独立翻译，结果一致
- ✓ 保持原有的上下文处理逻辑
- ✓ 错误时自动降级到串行模式

### 内存使用

并行模式会：
- 增加内存使用（需同时处理多个批次）
- 增加网络连接数

通常影响很小，但在资源受限环境下需注意。

## 🔍 故障排除

### 问题1：并行速度提升不明显

**可能原因**：
- API响应慢（网络问题）
- MAX_WORKERS设置过低
- 文本数量太少

**解决方案**：
```env
# 尝试提高并发数
MAX_WORKERS=8
```

### 问题2：经常出现翻译失败

**可能原因**：
- 触发API速率限制
- MAX_WORKERS设置过高

**解决方案**：
```env
# 降低并发数
MAX_WORKERS=3
```

### 问题3：想完全禁用并行

**解决方案**：
```env
# 关闭并行翻译
PARALLEL_ENABLED=false
```
