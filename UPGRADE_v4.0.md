# 升级指南 - v4.0

## 🎉 重大更新

### v4.0 新特性

1. **全新UI界面** 🎨

   - 更多并发批次，更快的翻译速度

4. **改进的错误处理** 🛡️
   - 更严格的AI返回结果解析
   - 解决"翻译数量不匹配"问题
   - 更智能的降级机制

## ⚡ 性能优化详解

### 1. 异步IO vs 多线程

**多线程问题**（v3.0）：
- Python GIL限制
- 线程切换开销
- 并发效率低（29.6%）

**异步IO优势**（v4.0）：
- 无GIL限制
- 事件驱动，开销小
- 并发效率高（60-70%）

### 2. 批次优化

**v3.0策略**：
- 批次大小：25
- 并发数：6
- 总批次少，并发利用率低

**v4.0策略**：
- 批次大小：15（更小）
- 并发数：12（更多）
- 总批次多，并发利用率高

**示例**：翻译120个文本
- v3.0：120/25 = 5个批次，6个worker，只用5个
- v4.0：120/15 = 8个批次，12个worker，用8个

### 3. 错误处理改进

**问题修复**：
```
警告: 翻译结果数量不匹配 (期望 25, 实际 46)
```

**原因**：AI返回了多行或额外内容

**解决方案**：
1. 更严格的提示词
2. 基于编号索引的解析（而非行号）
3. 验证每个编号的有效性
4. 自动过滤无效结果

## 🔧 故障排除

### 问题1：ttkbootstrap安装失败

**解决方案**：
```bash
pip install --upgrade pip
pip install ttkbootstrap
```

### 问题2：界面显示异常

**可能原因**：主题不兼容

**解决方案**：
在 `main_window_new.py` 第16行，更换主题：
```python
super().__init__(themename="cosmo")  # 或 "litera", "minty"
```

### 问题3：异步翻译报错

**可能原因**：事件循环冲突

**解决方案**：
程序已自动处理，如果仍有问题，在 `.env` 中设置：
```env
PARALLEL_ENABLED=false  # 临时禁用并行
```
